简介

grpc提供两种认证方式：1.SSL/TLS认证方式， 2.token认证。grpc也提供接口自定义认证。未进行认证的rpc方式进行通信是明文传输的，非常不安全。未使用认证的rpc请求使用grpc.WithInsecure()选项来跳过认证。

拦截器

简单来说，拦截器就是服务端进入请求处理函数之前或者客户端执行发起请求之前的处理逻辑。比如：grpc服务端进入请求处理函数前对请求进行校验，如果校验不通过就直接返回错误，不进入实际处理函数。拦截器分成普通方法拦截器和流式方法拦截器。

服务端拦截器

函数原型：func(ctx context.Context, req interface{}, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (interface{}, error)
前面两个参数是普通的grpc处理函数的参数，第三个包含处理函数的一些信息，第四个是处理函数。
grpc.ChainUnaryInterceptor函数接收多个拦截器实体，它们之间呈链式形式。最先传入的拦截器处于最外层，最后一个拦截器才是真正会调用处理函数。

客户端拦截器

客户端拦截器原型：func(ctx context.Context, method string, req, reply interface{}, cc *grpc.ClientConn, invoker grpc.UnaryInvoker, opts ...grpc.CallOption) error
