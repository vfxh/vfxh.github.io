内存对齐

请设置文章作者
Golang

发布于：Jun 5, 2020
次浏览

C语言可以用offsetof函数来获取结构体成员的偏移量

基于Fedora32 64位系统

简介

现代计算机的内存空间是按照字节来划分的，但计算机系统对于基本类型的数据存放在内存的地址有限制。计算机系统会要求这些数据的存放起始地址是某个数的倍数（2、4或8等等），这个就是内存对齐。而CPU在访问数据时候，并不是按照一个字节一个字节来访问的，而是按照多个字节组合作为访问单位来周期性性访问。这个多个字节组合一般是：2个字节、4个字节、8个字节甚至是16，32个字节。而这些访问的单位被称为内存存取粒度。

CPU存取示意图

为什么要对齐

假设CPU存取粒度为4，要读取一个int32（4个字节）的变量，根据内存存取粒度为4的情况，每一个周期读取数据都要读取4个字节。假设现在没有内存对齐，并且这个int32类型的变量数据随意存放，假设它的存储起始地址是1，那么这个数据占据的内存地址就是1，2，3，4。现在要从0地址开始读取，一个周期读取4个字节，包括0，1，2，3。但是int32变量的数据还包括地址为4的数据，那么就需要进行迭第二个周期读取，读取包括4，5，6，7。然后将不需要的字节剔除，就是把0，5，6，7这个四个地址的字节剔除，最后剩下的1，2，3地址的数据块和地位是4的数据块组合得到int32变量的数据。

没有内存对齐读取数据示意图

从示意图上看，没有内存对齐机制。那么在存取粒度是4的情况下读取一个int32变量的数据，大概需要3个步骤。1. 读取前一部分数据，剔除不需要的数据。2. 读取后一部分的数据，剔除不需要的数据。3。 合并两个数据块，合成需要的最终数据块。

而有了内存对齐机制之后，数据就只能按照对齐规则存放在内存当中。CPU读取数据的时候就可以一次性读取全部数据，就不会有额外的操作，提高了效率。比如上面的例子，CPU按照存取粒度为4读取数据，一个周期读取4个字节。而int32变量刚好是4字节，在内存对齐的机制下，这个变量的存储地址只能是4的倍数，所以CPU一个周期就可以全部读取出来。

内存对齐规则

C语言部分

不同平台有不同的默认内存对齐值，所以一般使用预处理指令*#pragma pack(n)*来指定按照n个字节对齐。n的取值有：1，2，4，8等等，也就是其指定的对齐系数取值是2^n，n取值>=0。其上限…我不知道。（应该是基础类型的最大类型长度，即使再往上也没有意义。当然，这是我猜的~~ ^_^）

结构体对齐规则

结构体有效对齐系数 = min[结构体成员中类型最大的长度, 指定对齐系数(或者默认对齐系数)]

  1. 结构体第一个成员的偏移量为0，之后的每个成员相对于首地址的偏移量都是该成员类型大小和有效对齐系数两者之间较小值的整数倍。如果有需要，成员之间会被编译器填充字节。
  2. 结构体的总大小为能够容纳所有成员数据的有效对齐系数的最小整数倍，并且按照有效对齐系数对齐。如果有需要编译器会在最后一个成员后面填充字节。

对于第二调规则的解释：

有一个结构体A，A作为另一个结构体B的内嵌成员。那么B的成员A相对于B的首地址偏移量就是：A的有效对齐系数和B的指定的对齐系数中较小的那个的整数倍

例程

在历程中，结构体struct inner指定的对齐系数是8，其成员类型最大长度也是8，所以struct inner的有效对齐系数就是8。因此，在结构体struct outside中，成员b相对于首地址的偏移量就是4的整数倍。因为struct outside的指定对齐系数是4，所以struct inner在struct outside的有效对齐系数就是4。

内存对齐示例

结构体例子

首先确定有效对齐系数：结构体的有效对齐系数是结构体中成员类型大小最大的值和指定对齐值（或默认对齐值）较小的那一个。结构体指定对齐值是4，结构体中类型最大的是int，也就是4，所以有效对齐系数是4。

分析

因为上面的结构体指定对齐值是4，而结构体类型长度最大的是int，也就是4。所以，有效对齐系数是4。
得到有效对齐系数之后就可以对结构体成员进行分析了：

  1. 对于成员a：根据规则1，a的偏移量是0，占第用0单元；
  2. 对于成员b：有效对齐系数是4，b自身的类型大小是4，所以偏移量是4的整数倍。所以b的起始地址从第4个单元开始，占用4，5，6，7四个单元；
  3. 对于成员c：有效对齐系数是4，c自身的类型大小是1，所以偏移量是1的整数倍。因此c的起始地址从第8单元开始，c占用一个字节，也就是第8单元；

这个时候需要注意，成员b占用的单元是从第4单元开始的。也就是说1，2，3这三个单元是空出来的，编译器需要对这三个单元进行字节填充。这样才能保证成员b是从第4单元开始，也就是偏移量是4的整数倍。
然后开始计算结构体整体大小。a成员占用一个单元，之后填充3个单元。b占用4个单元，不需要填充。c成员占用一个单元，一共是9个字节。因为有效对齐系数是4，所以结构体也按照4字节对齐。根据结构体的总大小为有效对齐系数的最小整数倍的规则，应该占用12个字节（这样才可以既符合结构体内存对齐，又能存储所有成员数据）。所以c后面需要再填充3个字节。

这个例子最后打印结果都是8，但是它们各自的内存布局是不同的。具体看图示：

到目前为止都是按照4个字节对齐，那么试试其它字节对齐如何，使用的还是上面第一个例程。

可以看到，指定8个字节对齐，最后得出的结果和指定4个字节对齐是一样的结果。其原因是实际的有效对齐系数并不是指定的对齐值的公式：

结构体有效对齐系数 = min[结构体成员中类型最大的长度, 指定对齐值(或者默认对齐值)]

结构体struct pkg的成员中类型最大的长度是int，也就是4个字节，也指定对齐值长度相比自然是4较小。所以有效对齐系数就是4，所以结构体struct pkg指定8个字节和指定4个字节对齐是一样的效果。
但是后面两个就神奇了，一个指定2字节对齐，一个指定1字节对齐。虽然a的值能够正确打印出来，但是b和c的值却没有打印出正确值，结构体的大小也不同。不用说，结构体的内存布局改变了，所以按照之前的对齐系数来获取数据就会出错。

内存布局示意图

现在分析一下

  1. 对于指定2字节对齐的情况，根据有效对齐系数的规则可以得出有效对齐系数是2，因为int的类型大小是4。4比2小，取较小值那就是2。a依然是相对于首地址偏移0的位置，占用一个字节，所以a占用了第0单元。而b是4个字节，根据原则1，b距首地址的偏移量是2的整数倍，所以是从第2个单元开始。因此第1单元为空，填充字节。又因为整个结构体的大小是能够容纳所有成员数据的有效值的最小整数倍，因此结构体大小为2*4，也就是8。这就需要c后面填充一个字节才符合内存对齐机制。
  2. 对于指定1个字节对齐的情况，其有效对齐系数是1。所以就不需要填充字节，直接按照结构体的各个成员对应的类型大小来存储就可以了。

不过对于1字节，2字节对齐这种情况，CPU取出数据就麻烦了很多。就和文章的开头的例子一样，b成员占用4个字节。存取粒度为1或者为2都需要多次读取数据才能够将一个成员的所有数据取出来，这样效率很慢。所以尽量指定合适的存取系数（粒度），这样CPU存取数据的时候一次就可以完成，从而提高了效率。

8对齐的例子，要涉及到有8字节的类型，这样才能判断结构体最大类型和指定对齐值的最小值来判断。

pkg2分析

先确定有效对齐系数。指定的对齐值是4，pkg2结构体成员中数据类型长度最大为double，也就是8。根据有效对齐系数原则，可知有效对齐系数是4。结构体第一个成员偏移量是0，a的类型是char，占一个字节。而b的类型是double，占8个字节。根据规则1，b相对于首地址的偏移量是有效对齐系数4的整数倍，也就是4。所以a和b之间需要填充三个字节。c的类型是4，所以c但相对于首地址的偏移量是有效对齐系数4的倍数，也就是从12开始。最后确定结构体的大小，也就是1+3+8+4 = 16，正好是4的倍数，所以不需要填充字节。

pkg6 分析

还是先确定有效对齐系数。指定对齐值是4，但是pkg6中最大类型长度是char的长度，也就是1。所以，有效对齐系数是1，而不管数组有多长。因此a，b，c三个成员都是紧挨在一起存储。最后结构体大小也是能够存储所有成员的最小有效对齐系数的倍数，也就是5（1*5）。

判断默认对齐系数的例子。如果默认对齐系数是4，那么就打印出16，如果是8，那么就打印出24。

不同内存对齐系数的内存布局

对于默认对齐系数是4的情况，其有效对齐系数是4。所以除了结构体第一个成员之外，之后的成员相对于首地址的偏移量都是4的倍数，所以a和b之间不用填充字节。而结构体的总大小也是4的倍数，所以需要在c的后面填充3个字节。
对于默认对齐系数是8的情况，其有效对齐系数是8。所以a和b之间需要填充4个字节，才能保证b的起始位置是8的倍数。同样c后面需要填充7个字节才能保证结构体是按照8来对齐的。

第二次修改分析
struct inner结构体成员d的类型修改之后，这个结构体的有效对齐系数就是4，而不是8了。因为结构体有效对齐系数是指定或者默认的对齐系数和结构体成员的最大类型长度两者之间的较小值，因为8比4要大，所以它的有效对齐系数是4。因此，struct inner结构体是按照4字节来对齐的。那么接下来就是确定struct inner在struct outside中相对于首地址的偏移量。由于struct outside指定的对齐系数是8，并且它的成员最大类型长度也是8（long），所以结构体struct outside是按照8字节对齐。但是struct inner结构体的有效对齐系数是4，根据规则1。struct outside成员b相对于首地址的偏移量就是4的整数倍。所以，struct outside成员a和struct inner成员b之间需要填充3个字节，才能符合内存对齐机制。

golang类型大小（基于64位平台）

类型 大小
bool 1 byte

总结

应该把类型长度较大的成员排在前面，减少字节填充的单元，这样就可以使得同一结构体的占用的内存较小。
