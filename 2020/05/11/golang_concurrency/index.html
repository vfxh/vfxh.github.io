Golang之并发

Posted on 2020-05-11 Edited on 2021-05-23 In Golang

简介

并发：逻辑上具备多个任务在一段时间同时执行的能力，从宏观的角度来看是同时执行的的，从微观的角度来看是串行的。
并行：物理上依赖多个处理器在某一个时刻真正意义上同时执行多个任务的能力。

并发执行

在普通函数或者方法前面加上关键字前缀go，可以让该函数或者方法在goroutine中调用，go语句本身执行立即完成。但是go关键字并不是并发执行操作，它的作用是创建一个并发执行单元。这个并发执行单元并不会立即被执行，而是在创建之后被放入一个系统队列，等待调度器取安排合适的系统线程去获取执行权。然后从队列中获取一个goroutine放到系统线程中执行。
一个执行单元处理保存函数指针和参数之外，还会分配执行所需要的栈空间。和系统线程相比，goroutine的栈栈空间是可以增长的。最初goroutine的空间很小，仅有KB级别。在扩容之后，可以增长到GB级别。另外，goroutine是由go本身调度的。
go 关键字作为一个匿名函数的前缀，表示这个匿名函数作为一个goroutine执行单元。这里不会阻塞，并且新创建的并发单元可能不会被立即执行。所以，需要等到一段时间。等到goroutine执行完毕之后才会退出main函数。否则，一般main函数结束之后。所有的goroutine都会被暴力终止。

对于多个goroutine，并且需要等待所有goroutine结束才退出，这个就需要sync.WaitGroup。sync.WaitGroup用于等待一组goroutine的结束。
Add()用于内部计数器加上指定的值，应该放在并发单元创建之前。Done()是递减内部计数器的值，应该在goroutine最后再调用。Wait()一直阻塞，至到计数器的值为0。如果计数器的值为负数，那么Wait()就会panic。

GOMAXPROCES

运行时可以创建很多个线程，但真正参与并发执行任务的线程有时候只有几个。默认线程的个数等于系统的处理器的核心数相等，可以使用runtime.GOMAXPROCES(n)函数来设置调度器使用的逻辑处理器的个数（同时执行任务的CPU个数）然后返回当前的设置值，如果。runtime.NumCPU()函数返回当前机器的CPU核心个数。

local storage

goroutine和系统线程不同，它没有返回值，无法设置编号，也无法设置优先级。不过除了优先级，其它可以通过代码来实现。
可以声明一个结构体数组，结构体成员可以声明自定义的goroutine编号，返回值等等。当然，这个也可以用map来实现，不过需要考虑到并发读写map的问题。

用结构体数组实现需要实现给出数组长度，用map实现的话需要考虑到并发读写的问题，同样slice也需要考虑到并发读写。

Gosched

调用Gosched函数的goroutine会暂停，让出处理器，处理器转去执行其它任务。当前任务会被放回队列当中，等待下次被调度恢复执行。由于goroutine执行的顺序是不确定性的，所以有可能goroutine b会先执行。

Goexit

Goexit被调用后立即终止当前goroutine，并确已经注册的延迟调用被执行。调用Goexit的goroutine不会影响到其它goroutine，也不会引起panic。

打印结果

如果再main中调用Goexit，会等到所有任务完成之后，再让进程崩溃。原因是：Goexit被调用之后，不会让main返回。因为main没有返回，所以main函数一直无法退出，形成死锁。等到其它任务执行完毕之后进程直接崩溃。

channel

简而言之，channel就是goroutine之间的一种通信机制，channel允许一个goroutine发送数据到另一个goroutine。channel是引用类型，创建的时候需要使用make函数来创建。channel传输的数据类型是需要指定的，比如传输int数据类型，那么这个channel的类型就是：chan int。channel支持三个操作，发送，接收和关闭。
关闭操作表示当前channel已经没有值在发送了，从一个关闭的channel读取数据，将会获取到channel里面的所有数据，直到为空。如果再从一个已经关闭channel读取数据将会读取到传输类型的零值。

channel类型

创建一个传输int类型数据的channel

ch := make(chan int)

这样不指定长度的channel类型称为无缓冲channel，与之相反的是有缓冲channel。创建一个容量为3的有缓冲channel。有缓冲channel可以用内置函数cap获取其容量，len函数获取当前channel里的数据个数。无缓冲channelcap和len都是0。

ch := make(chan int, 3)

单向channel类型
<-在chan左边是一个只能接收的channel，不能发送。而<-在chan的右边则是一个只能发送不能接收的channel。
比如下面的ch是一个只能接收的channel类型，test函数里只能从ch中读取数据，而不能向ch发送数据。

func test(ch <-chan int) {…}

一个只能发送数据的channel声明

func test(ch chan<- int) {…}

channel的缓冲区大小和类型无关，比如make(chan int)和make(chan int, 4)创建的channel类型都是chan int。channel是引用类型，可以用==来判断两个channel对象是否引用同一个底层对象，或者和nil比较判断是否已经初始化。

未初始化的channel进行写操作会panic，进行读操作会一直阻塞。关闭一个未初始化的channel会panic，重复关闭一个channel也会panic。

无缓冲channel

无缓冲channel也成为同步channel。使用无缓冲channel时，发送方发送送数据到channel中，如果接收方没有接收数据。那么发送方就会一直阻塞，直到接收方从channel中读取数据，然后接收方和发送方继续执行。如果接收方先执行，执行读取操作的时候发送方还没有向channel发送数据，这时候接收方就会阻塞，直到发送方发送数据。这时候双方才会继续执行下去，这也是无缓冲channel被称为同步channel的原因。
使用无缓冲channel的时候需要有配对的goroutine来执行发送和接收操作，如果只有一个goroutine，那么将会一直阻塞，形成死锁。

单向channel
channel默认是不区分读端和写端的，不过也可以声明成读类型的channel或者写类型的channel，这种区分读写端的channel称为单向channel。
声明一个单向读channel

var rch <- chan int
声明一个单向写channel
var wch chan<- int
当然也可以用make创建一个单向channel，不过没有意义。比如创建一个单向写channel，那么只能写，不能读，那有什么意义?

可以用make创建出channel，然后再赋值给单向类型的channel，或者传给单向channel的形参

单向channel无法重新转换成不区分读写端的channel

ch = (chan int)(rch) // cannot convert rch (type <-chan int) to type chan int
close可以关闭写端，但是不能关闭读端。原因是：关闭写端，读端还可以从channel里读取剩余的数据，如果没有数据就会返回对应类型的零值，而且可以判断channel是否已经关闭了。但是关闭读端的话，写入channel的数据就没办法获取了，会一直存在，浪费资源。（我猜的！真是情况学到源码分析再改！）

有缓冲channel

有缓冲的channel有一个元素队列，在创建channel的时候可以指定这个队列的长度。在执行发送操作的时候就把数据插入到队列的尾部，读取数据就从队列的头部弹出一个元素。当channel满的时候，执行发送操作就会阻塞，直到队列有空位。而当队列为空的时候，接收操作也会阻塞，直到队列有元素。

有缓冲channel不会像无channel那样如果没有配对的接收goroutine，执行写操作就会阻塞。对于有缓冲通道，如果channel的缓冲区没有满或者数据没有读取完之前，无论是读写操作都不会被阻塞。

上面的程序不会阻塞，因为写入三个数据之后channel的缓冲区才满，等到第四次写入并且没有读取数据的操作，那么就会阻塞。
有缓冲channel也叫异步channel，可以根据它的特性来写异步程序。多数时候，有缓冲channel可以提高性能，减少排队阻塞。

goroutine泄漏

看一个例子
test函数中创建了两个goroutine，以及一个无缓冲channel。goroutine b先执行，向channel发送事件通知，然后b退出，test的channel接收数据然后返回。而goroutine a在睡眠结束后向channel发送事件通知（channel被引用，延长了生命周期），然而这是一个无缓冲channel，test函数已经返回了，没有可以接收数据的goroutine，所以goroutine a就一直阻塞在发送数据这里，一直到进程结束。这种情况就称为goroutine泄漏。GC并不会回收一直阻塞的goroutine，所以这些泄漏的channel就会一直存在并且处于阻塞阻塞状态，占用资源。

解决上面程序中goroutine a一直阻塞的方法就是使用有缓冲channel。这里的有缓冲channel即使容量只有1，在test从channel读取出数据的时候，channel就变成空的了，所以goroutine a向channel写入数据也不会阻塞。

select多路复用

在处理多个channel的时候就需要用select，select的每一个case处理一个channel，选择已经准备就需要的channel并执行相应的代码块。如果存在多个case都符合情况，那么就随机选择一个执行。如果都不符合，那么就会执行default。

打印结果

一个例子，这个例子会打印出：0 2 4 6 8
原因是：对于一个容量是1的有缓冲channel，如果channel未满，那么写入数据不会阻塞，但是读取数据会阻塞。所以，当第一次循环的时候，只有ch <- i满足，写入0。但是第二次执行的时候，i的值为1。这个时候，channel已经满了，ch <- i会阻塞，而x := <-ch满足条件，就会执行这个条件对应的代码块。就这样依次循环，直到循环结束。

但是出现channel既没有满，也不是空的情况。那么就会出现随机输出的情况，因为select对于多个条件满足的时候就会随机选择一个，至于选择哪一个是不确定的。
