数据库之关系数据模型

Posted on 2020-05-22 Edited on 2021-05-23 In 数据库

概念

数据库模型分为很多种，关系模型是主要的数据库模型。除此之外，还有半结构化模型，对象关系模型，面向对象数据库模型。还有已经被淘汰了的基于树结构的层次模型和基于图结构的网状模型。
一个关系的例子：

关系：

表示信息的二维表，包括表的结构和表存储的数据

属性：

关系的列就是属性
比如例子中的属性就是：title、year、length、genre

模式：

关系名和属性的集合组合在一起称为关系模式
在关系型数据库中，数据库就是一个或多个关系组成的。关系模式的集合就叫做关系数据库模式，也叫数据库模式
例子中的模式可以表示为：Movies(title,year,length,genre)

元组：

就是关系中除了属性之外的一行记录值。
每一个元组有若干个分量，分量就是元组中对应的某个属性的值。比如例子中的关系有两个元组，第二个元组有四个分量two、1999、123、B，分别对应着四个属性，依次是title、year、length、genre。元组的表示：
(two,1999,123,B)

域

属性的数据类型称为域
在加上域之后，例子中的关系模式就可以表示为：Movies(title:string,year:integer,length:integer:genre:string)

关系实例

关系模式和元组集合组成的实体
关系实例不会一成不变，而是动态的，一个关系可能会进行增加元组或删除元组，或者是更新元组的分量等操作。关系的模式也可能会改变，比如增加属性或者删除属性。但是模式的改变通常代价很大，假设一种情况：关系的元组非常多，数以百万计。增加或删除属性的时候就通常会伴随着数百万个元组的分量增加和删除

键

键是由关系上的一个或多个属性组成的约束
键的作用是保证在一个关系中不同的元组之间在声明为键的属性上的值不同，也就是说键的作用是保证元组的唯一性

  1. 比如关系Movies，假设title属性的分量是具有唯一性的，也就是关系Movies的所有元组在属性title的值都不同。那么就可以把title声明为键，即使其它属性的分量可能相同，也能根据title的分量来唯一标识每一个元组。
  2. 又假设title可能会相同，year也可能会相同。但是title和year的组合是不同的，具有唯一性。那么就可以把title和year声明为键，这样title和year就可以唯一标识每一个元组，保证了元组的唯一性

SQL中定义关系模式

SQL是结构化查询语言Structure Query Language的缩写，是一种和数据库管理系统进行交互的语言。SQL分为三种子语言，分别是：

  1. 数据库定义语言，用来对数据库模式操作，比如：CREATE, DROP, ALTER
  2. 数据库操纵语言，用来对数据库数据进行增加，删除，查询，更新等操作，比如：INSERT, DELETE, SELECT, UPDATE
  3. 数据库控制语言，与数据的安全、状态有关的操作，比如：GRANT, REVOKE, COMMIT, ROLLBACK

SQL中的关系分为三类：

  1. 表：真正存储在数据库中的一种结构化数据结构，可以对其元组进行更新删除等操作。
  2. 视图：视图时一种虚拟的表，可以引用一个或多个表。然后对它进行查询操作，就像从真正的表中查询一样。但是视图并不存储在数据库中，也就是不以物理的形式存在。
  3. 临时表：由SQL处理程序临时创建的，临时表在结束之后就会连带着数据一起被删除，并不会存储在数据库中。

代数查询语言

关系代数的操作主要有四类：

  1. 集合操作：并，交，差
  2. 消除列的投影操作和消除行的选择操作
  3. 重命名操作，对查询结果重命名，临时修改结果关系的属性名，不影响原始关系属性
  4. 组合两个关系的操作。笛卡儿积运算尝试两个关系所有元组的配对，组成一个结果关系。联结操作，配对符合要求的一些元素得到一个结果关系。

集合操作

给定集合R = {1,2,3,4,5}，S = {1,4,6,7,8}：

  1. 并操作：R ∪ S，得到的结果关系中的元素来自R或者S。也就是说，结果关系中的元素要么在R中要么在S中。如果一个元素在R和S中都存在的情况下，该元素只在结果关系中出现一次（集合不存在具有重复元素的情况，所以就不存在一个集合中多个重复元素是否只出现一次的问题）。所以R ∪ S的结果就是：{1,2,3,4,5,6,7,8}。
  2. 交操作：R ∩ S，得到的结果关系中的元素是R和S共同存在的元素。因此，R ∩ S的结果是：{1,4}。
  3. 差操作：R — S，得到的结果关系中的元素是在R中存在，但是在S中不存在的元素。需要注意得是，S — R和R — S是不同的。S — R表示的是最后的结果关系的元素是在S中存在，但是在R中不存在的元素。S — R和R — S的含义截然相反。根据定义，R — S的结果是：{2,3,5}，而S — R的结果就是：{6,7,8}。
