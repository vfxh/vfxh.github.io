栈和队列

Posted on 2020-05-22 Edited on 2021-05-23 In 数据结构

栈

栈是一种先进后出的数据结构，入口和出口都在同一端，称为栈顶，所有操作都在栈顶进行。例如：进栈顺序是：1 2 3 4。出栈的顺序就是倒着过来：4 3 2 1。
栈模型

一个空栈模型
（这里的栈顶和栈底和操作系统里的栈顶和栈底不是一个意思，这里额栈顶是指入栈和出栈的一端，栈底是另一端）

用top指向最新的元素位置，因为这是一个空栈，所以top指向栈以外的地方

入栈（压栈）

新的元素入栈之后，top向栈顶方向移动，指向新的元素的位置，这样最新入栈的元素就是栈顶元素。

出栈

元素出栈会从栈顶弹出，然后top向栈底方向移动一个位置

队列

队列是一种先进先出的数据结构，入口和出口在两端。入口端只能入队，出口端只能出队。一般把出队的一端称为队头，入队的一端称为队尾。如果压栈的顺序是：1 2 3 4，那么出栈的顺序就是：1 2 3 4

rear指向下一个空位，给新的元素执行位置

满队

根据当前实现，当队列元素个数达到len(array)-1时，就判定队列已满

出队

出队一个元素，这个时候队列已经有空位了。但是rear指向数组的最后一个位置，如果再递增rear就会出现数组越界，程序错误。所以，需要把rear指向新的数组的开始处，形成一个循环队列。新rear的值用一个公式得出：(rear+1) % len(array)，并且当(rear+1) % len(array) == front时就表示队列已满。
对于m和n都都不为负数的情况下，可以有以下规律：
有x = m % n，则：0 <= x < n, x = m - nc, 其中c大于等于0的整数
比如：0 % 6 == 0，1 % 6 == 1 … 6 % 6 == 0, 7 % 6 == 1 依次循环

再入队一个元素

这个时候队列已满，由上面的公式：(rear + 1) % len(array) == front，代入rear和front的值就是：(0 + 1) % 6 == 1，所以队列满

为什么需要留出一个空位？

根据当前的实现，需要根绝上面给出的公式来判定队列是否已经满的原因。当然不用这种方式也可以，后面再讨论。

初始化

初始化的时候给结构体的数组分配合适的内存，然后将rear和front两个成员都赋值0，表示队列为空。len表示当前的元素个数，也初始化为0。cap表示数组的容量，赋值给定的count。

入队

在更新rear成员的值的时候，不能直接递增rear的值。队列不涉及循环的时候没什么问题，但是涉及到循环队列的时候必须进行求模运算，让rear的值达到数组的最大长度的时候从0开始。

出队

根绝当前的思路和实现，rear和front值相等的时候就是空队列。更新front的值的时候也需要根据公式来计算，让front值能够循环。

初始化

初始化把Queue的各个成员都设置为空，这个时候表示队列的链表没有

入队

先判断队列是否为空，如果为空，那么front和rear指针都指向这个节点。如果不为空，那么就把新节点插入链表尾部，rear指向它。下面是队列为空的情况模型，链表不为空的情况上面已经提到了。

出队

出队的时候首先要判断是否为空队，如果不为空队就正常删除链表的头节点，然后front指针指向下一个节点。但是队列只有一个节点的时候，删除这个节点之后front和rear指针都指向NULL。当前实现front和rear指向相同有两种情况，一个是空队，一个是只有一个节点，所有需要进一步判断。

总结

栈就是先进后出的数据结构，队列就是先进先出的数据结构。理解这两种数据结构还是蛮重要的，对于语言底层原理的理解和其它数据结构的使用都有意义。比如golang中的defer特性，就是和栈一样的特性。同一作用域下的多个defer声明的执行顺序和栈一样，都是先进后出的。二叉树的非递归遍历也需要用到栈和队列。
