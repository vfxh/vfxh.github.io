Linux之用户管理和文件权限

Posted on 2020-05-14 Edited on 2021-05-23 In Linux

简介

Linux是多用户系统，不同的用户相互隔离。对于Linux的文件，可以赋予不同的权限，来决定不同的用户是否可以对文件进行操作。

用户管理

在用户创建的时候，系统会分配给新创建的用户一个唯一的ID，称为UID。系统管理用户权限是根据UID来跟来跟踪，不过在用户登陆的时候使用的是用户名。用户信息保存在/etc/passwd文件。这个文件的每一行记录对应一个用户信息，比如root用户，它的UID是0，是Linux系统的超级用户。每一条记录有七个字段，对应着不同的信息，分别是：用户名:密码:用户ID:组ID:备注:HOME目录:默认登陆shell。以root用户为例：

对于密码字段，所有的用户密码都被设置为x，而真正的用户密码存储在另一个文件/etc/shadow，只有root用户才能访问

添加用户

添加用户使用的命令是：useradd，创建一个用户需要带上可选的命令参数和必须的用户名。如果用户名已经存在，那么就会报错：user ‘username’ already exists。可以用id命令来判断一个用户是否存在，id命令的作用是打印出指定用户名的真实和有效的用户ID以及组ID。

#表示超级用户，”表示注释

useradd 命令在创建用户的时候，会根据命令参数和系统的默认值来设置用户。使用-D参数可以查看系统的默认值。
