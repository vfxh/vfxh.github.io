Golang之module使用

2020-08-02 Golang

Contents

  1. 介绍
  2. 使用
    1. 创建
    2. 依赖管理

介绍

module是go1.11版本开始引入的包管理工具，解决依赖的问题，可以在$GOPATH/src以外的地方存放源文件，而不用担心路径找不到的问题。

使用

创建

进入$GOPATH/src以外任意的空目录中，执行命令go mod init [module_name]，创建一个module。比如在路径D:\workSpace\learn_module下，执行go mod init example命令，就会创建出一个module，其名称就是example。这些信息在go.mod文件中，go.mod文件是自动生成的。

添加依赖
比如在main.go文件中引入gorm包，这个包目前是不在当前module里的。在对当前程序进行构建的时候，会自动去寻找并下载这个包。

执行结果

go.mod内容

这个包的源码会保存到$GOPATH/pkg/mod路径下。比如gorm.io/gorm包，它的源码会保存在$GOPATH/pkg/mod/gorm.io路径下。这个路径下的gorm包可能会有多个版本，不同版本的包会在不同的目录中，目录名称后跟着版本号。

go list -m 命令用于查看当前module名称。加上选项all是查看当前包名以及相关依赖信息。
