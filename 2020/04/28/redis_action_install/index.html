Redis实战笔记《一》：Redis安装和数据类型

Posted on 2020-04-28 Edited on 2021-05-23 In Redis

Redis不同MySQL这样的关系型数据，Redis是一种非关系型键值数据库，运行在内中。虽然可以做持久化设置，不过一般都是缓存来使用。因为是运行在内存中，并且在发送指令的时候没有经过查询分析器或者查询优化器的处理，所以Redis对数据的读取速度要高于MySQL这种关系型数据。另外，Redis对事务的支持没有MySQL那么完全，对于事务仅仅是不完全的支持。

二、数据类型
1. 键的操作命令


2. 字符串
增加字符串或者更新字符串的值
如果键不存在就创建，如果键已经存在了，那么就会把str_value设为当前str_key的值。值不限于字符串，也可以是整数，浮点数

获取字符串的值

删除字符串跟删除键是一样的操作
可以删除多个字符串，不过指定的键需要用空格隔开。返回的是成功删除的个数，如果键不存在，就返回0

3. 列表
列表可以有序存储多个字符串，对，它是有序的
向列表增加元素，如果列表不存在就创建这个列表

获取给定范围的列表的值
格式是：LRANGE [key_name] [start] [stop]
注意点：
1. start 是开始的索引，stop是结束的索引
2. 如果stop是负数，那么最后一个元素的索引是-1，如果stop过大，超过列表的长度，那么就返回：(empty list or set)
3. 如果stop是正数，并且超过列表的长度，那么就返回仅有的全部元素
4. 如果start和stop相同，那么就返回这个索引元素的值

弹出元素
1. 执行弹出元素操作之后，就从列表里面删除，并返回删除的元素值
2. 如果列表不存在，就返回(nil)

按索引查找元素
1. 如果键不存在或者索引超过列表长度，就返回(nil)

4. 集合
集合和列表都一样可以存储多个字符串，但不同的是：列表可以存储多个相同的元素，但是集合只能存储不同的元素。并且集合存储的数据是无序的，它底层是散列表实现，但只有散列表的键，没有相应的相关值。
增加元素
1. 如果键不存在就创建并添加元素，返回成功添加元素的数量
2. 如果添加的元素有重复的，那么就只增加一个，其余重复的元素忽略点
3. 如果元素已经存在，那么忽略掉增加的元素

删除元素
1. 如果元素存在集合中，那么删除并返回1
2. 如果元素不存在，那么就返回0

获取全部元素
如果集合没有元素或者不存在就返回：(empty list or set)，反之返回所有元素

判断元素是否是集合的成员
如果是则返回1，不是返回0，键不存在也返回0

5. 散列
散列底层数据结构是散列表，散列类型的对象的元素是一个个键值对。元素的值可以是字符串，也可以是整数，浮点数。
添加元素
如果元素存在，则返回1，反之返回0，更新数据也返回0

获取元素
如果元素不存在就返回nil，反之返回元素的值

删除元素
1. 可以删除多个元素，返回成功删除的元素个数
2. 如果元素不存在或者散列对象不存在则返回0

6.有序集合
有序集合和散列一样，都是用来存储键值对。不过有序集合的键是唯一的，不存在相同的键，这点和集合一样。有序集合的键也成为成员，对应的值称为分数，分数必须是浮点数。可以根据分数来排序。
添加元素
1. 虽然有序集合的成员是不能相同的，但是分是是可以相同的
2. 一次性可以添加多个元素，返回成功添加元素的个数。另外，分数在成员的前面

获取元素

根据分数范围获取元素
1. 如果分数范围内没有成员，就会返回：(empty list or set)
2. 如果终止分数超过所有元素的分数，那么就按其开始分数到最大分数的范围返回

删除元素
1. 如果元素不存在则返回0，对象不存在也返回0
2. 删除成功返回删除成功的元素个数
