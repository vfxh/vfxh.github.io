树不同于链表这样子的线性结构，只能一对一或者一对零。树是一种非线性结构，一个结点可以对应一个结点，也可以对应多个节点，甚至可以对应零个结点。在数据结构中，树的定义是：有n（n>=0）个节点的有限集合。当n等于0的时候称为空树，在一颗非空的树由如下特点：

  1. 有且仅有一个特定的，称为根的节点
  2. 当n>1的时候，其余节点可分为m（m>0）个互不相交的有限集合，每一个集合本身又是一棵树，称为子树

树的示意图

上图中，1是这棵树的根节点。根节点下有两个子节点，分别是2和3。没有子节点的节点称为树的叶子节点，在上图中，5，6，7，8，9这五个节点都没有子节点，所以是叶子节点。另外，根节点1的子节点2下面有很多个节点。它们共同的祖先都是2，如果以2为根节点，那么又是一棵树。称为根节点1的子树。当然，根节点1的另一个子节点3也可以作为子树的根节点，不过这棵树除了根节点3之外只有一个叶子节点。

从根节点到叶节点分成不同的层级，就像下图所示。树的层级从根节点开始，根节点所在的层级为第一层，它的直接子节点为第二层（如下图的2和3节点所在的层级），依次类推。所以，下图中的树一共有5层。对于一颗树的任意节点n，从根节点到n节点的唯一路径长称为n节点的深度。比如下图中的节点2的深度为1，10的深度为4。而根节点的深度为0。还是对于一棵树的任意节点n，以节点n为根节点的树，从n到属于这个树的叶子的最长路径长称为n节点的高度。比如节点2，它到叶子节点的最长路径长是3，也就是节点2到节点10的路径长。所以，所有叶子的高度都为0。因此，一棵树的高度等于它的最深叶子的深度。又是对于一个树的任意节点，它直接衍生出来的直接下一层级的节点称为该节点的子节点（除了叶子节点），比如节点4和5是节点2的子节点。而衍生出该节点的直接上一层级的节点称为该节点的父节点（根节点除外），比如节点1是节点2的父节点。同一层级具有共同父节点的节点称为兄弟节点，比如节点4和5有共同的父节点2，或者节点7，8，9有共同的父节点4，它们都是彼此兄弟节点。

二叉树

所谓二叉树就是每个节点至多只能有两个子节点，也可以有一个子节点或者一个子节点都没有。根据树的定义，没有子节点的节点称为叶子节点。一棵二叉树如下图所示。其中，每个节点至多只有两个字节。比如，根节点下有两个子节点，分别是2和3。2被称为根节点的左儿子，3被称为根节点的右儿子。以2为根节点的子树称为根节点1的左子树。

二叉树示意图

满二叉树，顾名思义就是所有非叶子的节点都有两个子节点，简而言之就是每个分支都是满的。

满二叉树示意图

完全二叉树指的是，最后一个节点之前的节点都是齐全的。有这样一个定义：对一个有n个节点的二叉树，按层级顺序编号，则所有节点的编号为从1到n。如果这个树所有节点和同样深度的满二叉树的编号为从1到n的节点位置相同，则这个二叉树为完全二叉树。有点虚，具体看下图。如果节点6补全它的右儿子节点，编号为13，那么这棵树也是完全二叉树。但是，如果节点6只有右儿子而缺少了左儿子，那就不能称为完全二叉树。因为节点11和节点13之间存在一个空位，缺了一个节点，不齐全。

存储结构

二叉树在内存中存储的结构有两种：一是链式存储，用链表实现。二是使用顺序存储，用数组实现。使用链式结构存储的二叉树每个节点至少有三个区域，与存储数据相关的区域，以及一个指向左儿子的指针域和一个指向右儿子的指针域。

数组存储示意图

在使用数组存储二叉树节点的时候，叶子节点，它的左右儿子所在的单元为空。所以，用数组存储普通二叉树会造成一定的空间浪费。用数组存储二叉树适合完全二叉树和满二叉树这样的特殊二叉树，因为它们的元素是连续的，不会造成中间数组某个单元出现空缺的现象。对于使用数组下标为0的单元存储节点的情况（也就是上图），对于一颗二叉树上的任意位置i，它的左儿子位于数组下标为2*i+1的位置上。它的右儿子则是位于数组下标为2*i+2的位置上。而这个i的父节点则是位于数组下标(i-1)/2的位置上（对于i所在的位置是右儿子，整数相除向下取整）。
不过，使用数组下标为0的位置存储二叉树节点的计算方式相对来说比较繁琐。将下标0的位置空出来不存储节点，其计算方式相对来说就简洁了很多。比如使用数组存储上面的完全二叉树，示意图就像下图这样。

二叉树的遍历

二叉树的遍历方式有四种，分别是先序遍历，中序遍历，后序遍历和层序遍历。先（前）序遍历指的是：先访问根节点，再访问左节点，最后访问右节点。中序遍历指的是：先访问左节点，再访问根节点，最后访问右节点。后序遍历指的是：先访问左节点，再访问右节点，最后访问根节点。层序遍历指的是：按照树的层次，从第一层级开始，依次往下。每一个层依次从左到右开始遍历，直到遍历完整棵树的方式。其中，层序遍历也叫广度优先遍历。而先，中，后三种遍历方式统称深度优先遍历。

层序遍历

层序遍历用队列来辅助，先将根节点入队。然后访问当前根节点数据，如果当前根节点的子节点不为空，那么就将子节点入队。这样，同一父节点的两个子节点总是左子节点的访问顺序优先于右子节点。因为父节点先入队，子节点后入队。父节点的访问顺序也优先于子节点，这就达到了从上之下，从左至右的目的。

其它知识点

已知两种遍历访问重建唯一二叉树

对于二叉树的遍历顺序有个挺有意思的知识点，就是 已知先序和中序，或者中序和后序可以唯一确定一棵二叉树的结构。但是已知先序和后序不能唯一确定一颗二叉树的结构。通常这个问题是这样的，给出先序和中序遍历顺序，求后序遍历顺序。或者给出中序和后序遍历顺序，求先序遍历顺序。比如：一颗二叉树，已知先序遍历的顺序是：ABCDEFGHI，中序遍历的顺序是：BDCEAGFHI，求这个树的后序遍历。

对于遍历顺序的，先序遍历的顺序第一个节点是根节点，后序遍历的顺序最后一个节点是根节点。根据这个特性，由此可以确定根节点。然后在中序遍历中找到根节点所在的位置，根据中序遍历的特性，位于根节点左边节点都是属于左子树的节点，位于根节点右边的节点都是右子树的节点。然后依次重复这个过程，直到确定最后的二叉树结构。

根据给出的先序和中序遍历顺序推导后序遍历，这个过程首先要根据给出的遍历顺序来重建一颗唯一二叉树，然后得出后序遍历，具体思路。

  1. 根据先序遍历的特性，可以得出A是根节点。然后就可以在中序遍历中找到这个根节点A，于是，左边的节点都是左子树的节点，如：BDCE。右边的节点都是右子树的节点，如：GFHI。这样，确定了根节点，也分离了左右子树。
  2. 分离了左右子树之后开始递归分离子树的子树，直到分离成单个节点。按照先序遍历的顺序，根节点A之后是B，可以确信B就是左子树的根节点。然后在根据中序遍历，可知B只有右子树没有左子树。然后再返回来先序遍历，B之后是C。所以B的右子树的根节点就是C，这样B的右子树就分离出来了。根据中序遍历顺序，得到D是C的左节点，E是C的右节点。
  3. 左子树构建完成之后开始分离右子树。根据先序遍历，根节点A的左子树遍历完之后的下一个节点就是右子树的根节点。所以，A的右子树根节点是F。然后在看中序遍历，得到G是F的左节点。HI是F的右子树节点。这个时候可以确定H是F右子树的根节点，I是H的子节点。那么I是H的右节点还是左节点呢。从先序是看不出来的，只能从中序遍历确定。根据中序遍历的特性，先左子树，再根节点，最后右子树。I排在H的后面，所以，I是H的右节点。

最终构建的二叉树如下面两张图图所示。根据构建出来的二叉树可知其后序遍历是：DECBGIHFA。

二叉查找树

二叉查找树有这样的特点，如果某个节点左右子节点都存在。那么它的左儿子节点的值小于该节点的值，它的右儿子节点的值则是大于该节点的值。所以，对于一颗二叉查找树的根节点，其左子树的所有结点的值都小于根节点的值，其右子树的值都大于根节点的值。***二叉查找树查找一个节点的平均时间复杂度是O(logN)***。

二叉查找树示意图

查找

根据二叉查找树的特性，查找一个节点相对于线性结构来说要快很多。比如链表，要查找一个特定的节点，即使该链表的每个节点的值都是经过排序的，它的查找平均复杂是O(N)。而二叉查找树则是O(logN)。当然，并非所有的线性结构都和链表一样，比如一个已经经过排序的数组，可以通过二分查找的方法来查找一个特定的元素，它的查找时间复杂度和二叉查找树的查找平均时间复杂度一样，都是O(logN)。

现在，我们来看一下二叉查找树的查找过程，比如查找值为4的节点，分成三个步骤。

  1. 首先将值4和树的根节点比较，4小于根节点的值6，根据二叉树的特性，查找的节点如果存在的话就在根节点的左子树当中
  2. 比较根节点的左儿子。发现4比这个节点要大，这个时候向这个节点的右子树接着查找
  3. 和它的右儿子比较，发现值相等，找到这个值为4的节点

查找过程示意图

实现这个查找算法需要稍微改一下原来的程序，虽然存储的数据使用字符也可以，但是没有整数来得直观。还得需要注意一点，创建函数并没有处理数据重复的代码。这么做的原因是这里只探索原理，不会太深入细节。所以，构建后的二叉树各个节点的值必须是唯一的，而且是有效的（为了区分空叶子节点）。

使用递归的方式实现不难，如果传进来的root为NULL，那么直接返回NULL表示el不存在。反之测试是否大于或小于当前节点值。如果大于，那么就递归查找右子树。如果小于，那么递归查找左子树。如果等于，那就说明找到该节点。即使元素不存在，当递归调用find函数最终root参数的值也为NULL。

插入

插入操作和比较操作有些类似，插入操作同样需要查找节点。不同的是，插入操作在未找到当前给定值的节点的时候，就创建一个新的节点作为树的新节点。还是上面的二叉树模型作为基础，比如插入一个新的元素5。那么插入操作流程就像下面这个样子：

  1. 将新元素值和根节点值比较，如果比根节点值小，那么就往左子树找空位。如果比根节点值大，那就往右子树找空位。依次递归，直到找到空位。
  2. 找到插入的位置之后，构建新节点，赋值。并且将其左右指针置空，因为是新节点，自然就没有子节点了。最重要是还需要将新节点的父节点指针指向它，否则这个新节点就找不到了。

如果想要存储多个相同的值，那么只需要在表示节点的结构体加一个用于统计相同值的字段即可。如果插入的值已经存在，那么就加1。在删除的时候就减1，等到这个字段值为0时表示当前节点值是最后一个，直接释放点这个节点的内存即可。

查找插入过程

那么，怎么让一个新插入的节点的父节点引用这个节点呢？一个可能的方式是：使用递归，返回这个节点的指针，然后父节点在保存这个指针的值到相应的指针域上。因为每一个节点都包含三个数据部分，数据域，左节点指针域和右节点指针域。

在下面这个例程种，先是判断根节点root是否为空。如果为空，那么就创建一个新的节点，并初始化赋值。这也就是为什么插入操作可以用于创建一颗二叉排序树的原因。之后的两个if语句分支是判断给定的元素值是否大于当前根节点的值（如果进入到子树，根节点就是子树的根节点），来决定像哪个子树查找空位。根据递归的特性，最后root的值必定为NULL（前提是创建的时候叶子节点的左右儿子指针为NULL，才能判断），那么就可以创建一个新的节点，然后返回这个节点的指针。
最后需要注意一点的是，必须返回节点的指针，否则无法引用新建的节点。最后调用insert函数的时候，也需要将原来的根节点变量用来接收insert函数的返回值，避免出错。如：root = insert(root, el);

删除

删除操作比较麻烦，需要考虑到删除一个节点之后如何维持父子节点之间相互关联，还有就是保证二叉树的特性：根节点的值大于全部左子树的节点值，根节点的值小于全部右子树的节点值。根据二叉树的特性，树的任意节点至多只有两个子节点。因此，删除节点可以分成三种情况：一是被删除的节点是叶子节点，二是被删除的节点有一个子节点（或左子树），三是被删除的节点有两个子节点（或两个子树，左右子树）。

对于叶子节点，直接删除就可以了。但是对于有一个子节点的节点，需要将父节点的指向绕过被删除的节点，指向被删除节点的子节点。这个过程在下图给出。

B-树

B-树是一种自平衡树，其每个节点可以有多个元素，如果每个节点至多只能由M个子节点，那么这个B-树称为M阶B-树。
M阶B-树的特点：

  1. 所有叶子节点都处于同一层
  2. 除了叶子节点之外，每个根节点至少有两个子树
  3. 非根非叶子节点有k-1个元素和k个指向子树的指针（m/2 <= k <= m），叶子节点只有k-1个元素
  4. B-树的结构在插入删除的过程中是动态变化的，知道操作完成之后才能确定B-树的结构。

查找
执行查找的操作首先从根节点开始，比较当前节点中的关键字。如果找不到就选择合适的指针域往下一个节点。因为节点中的关键字是有序的，可以使用二分查找等查找算法来较少时间。比如查找24：从根节点开始，可以确定在右子树中。又因为24比41小，可以确定在当前根节点的左子树中。最后在叶子节点中找到24。当前的B-树只有3阶，顺序查找和二分查找没什么分别。但是阶很大的时候，二分查找的优势就出来了。

插入
由于B-树的性质，插入操作就比较麻烦。在插入节点的时候，需要调整B-树使得保持平衡又不破坏B-树的性质。一种常规的做法是：分裂字节。如果一个节点元素已经满了，那么就需要分裂这个节点。使得其父节点多出一个子树。如果父节点的指针域也满了，那么就继续分裂，一直到树的根节点。当树的根节点也满了，那么就需要构建新的根节点。简而言之，B-树的这种插入方法就是自底向上构建树。直到所有插入操作完成之后，才能确定根节点的内容。

当然插入并非只有分裂这一个方法。另一种可信的方法是：在还有空位的情况下，调整关键字的位置，将原本已经满了的节点的其中一个关键字移动到另一个节点的空位当中。比如要插入60，就将57移动到53所在的节点当中。然后将58移动到原来57的位置，接着移动59代替58原来的位置。这样就可以空出了一个可以插入60的位置了。

B-树查找的时间复杂度是O(logM)，但是插入和删除需要O(M)的时间复杂度。

需要注意的一点事：以3阶B-树为例，小于k1的子树所有关键字都小于k1。大于关键字k1的子树所有关键字都大于k1。这样才能保证树的叶子节点关键字是有序的。

删除
删除事插入的反操作，插入需要分裂节点，那么删除就需要合并节点。

B+树

B+树是B-树的一种变体，具有比B树更高的查询性能。其性质和B树差不多，不过在此基础上增加或作稍微修改。
M阶B+树的特征：

  1. 有k个关键字的节点有k个指向子树的指针（B-树是k-1个）
  2. 非叶子节点不包含数据（B树的节点都可以包含数据），只用来索引，所有数据保存在叶子节点（或者指向数据项的指针）
  3. 所有中间节点的关键字都存在于直接子节点，在子节点中是最大或最小关键字
  4. 叶子节点之间通过指针串联起来

（争议：B+树可以有k个关键字，也可以有k-1个关键字）
