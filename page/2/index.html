Golang之channel和select

Posted on 2020-07-21 Edited on 2021-05-23 In Golang

channel

和slice，map等类型一样，channel也是一种引用类型，其主要用于goroutine之间的通信。一个channel类型变量引用一个runtime.hchan底层结构，这个结构是channel运行时的表示。

数据结构

在创建一个channel的时候，如果是无缓冲channel，比如：ch := make(chan int)，那么runtime.hchan.dataqsiz为0，runtime.hchan.buf为空指针。如果是有缓冲channel，比如：ch := make(chan int, 1)。那么runtime.hchan.dataqsiz则是1，runtime.hchan.buf则是指向一块能够容纳1个int型变量的内存。这块内存实际用于数组，为了能够复用空闲的位置，这个数组被设计成循环队列。不过创建有缓冲channel却不分配缓冲区内存的情况也有，比如元素类型是struct{}，这个就不会创建缓冲区。 如果忽略元素类型为struct{}这种不占用内存的元素类型，可以总结为：有缓冲和无缓冲的channel实际区别就是，在创建一个channel的时候是否有缓冲区，也就是用于缓存待发送或接收元素的循环队列。当然这个总结有瑕疵，但总体大致是这样。

数据库之约束和触发器

Posted on 2020-07-18 Edited on 2021-05-23 In 数据库

所谓约束，就是数据库中的一张表或者多张表之间存在着限制条件。对表的操作必须在限制条件的允许下才能正常执行，如果违反了限制条件，那么对数据库的操作将不能完成执行。比如建表的时候声明某一个字段为主键来保证表中每一行记录的唯一性，如果在这被声明为主键上的字段值重复了，就会出错。因为这违反了主键约束，不能够保证记录的唯一性。

外键约束

键是用于保证表中每一行记录的唯一性，是由一个或多个属性组成。如果是有过个属性组成，其中的某个属性在多行记录可以有相同的值。但是这些属性的组合必须是唯一，否则不能声明为键。键可以是主键（PRIMARY KEY）或者唯一索引（UNIQUE INDEX），唯一键（UNIQUE KEY）。主键和唯一索引或唯一键都是约束，这个约束是保证列值的唯一性（单个列值或多个列值的组合）。

主键和索引的区别：主键主要的作用是保证记录的唯一性，声明为主键的属性不能为NULL，也就是说不可以有NULL值。和普通索引一样，主键也可以增加检索的速度。唯一索引的作用和主键一样，都是用于保证表中每一行记录的唯一性。不同的是，声明为唯一索引的属性可以有NULL值，但是主键不可以。

外键指的是一张表中的属性引用另一张表中被声明为键的属性，被引用表的属性必须声明为键，否则不能作为外键声明。并且这张表中的外键属性值必须出现在另一张表被引用的属性中，否则会破坏引用的完整性。

Golang之测试

Posted on 2020-07-04 Edited on 2021-05-23 In Golang

单元测试

规则：

  1. 测试代码必须放在当前包以“_test.go”结尾地文件中
  2. 测试函数以Test为函数名前缀
  3. 测试命令go test忽略以“_”或“.”开头地测试文件
  4. 正常编译（go build/install）会忽略测试文件

testing是工具链和标准库自带地单元测试框架，编写测试包含单元测试函数地时候需要导入这个包。testing包中包含一个T类型，用来控制测试地结果和行为。比如根据测试例程来判断是否测试通过，来决定是否继续执行测试函数等。

测试例子

运行测试，-v选项可以输出包中每个测试函数的名称已经执行的时间。go test -v默认执行当前包中的测试函数，如果要测试当前包和子包，就执行 go test -v ./… 。不过我觉得没必要这么做，防止混乱。

testing.T.FailNow方法表示，如果测试失败就立即终止当前测试函数，不再继续测试当前函数的后面代码。testing.T类型的方法有很多，主要有几个。

方法 说明
FailNow 失败，立即终止执行当前测试函数
Fail 失败，继续执行当前测试函数
SkipNow 跳过，停止执行当前测试函数
Log 输出错误信息，仅失败或-v时输出
Parallel 和有同样设置的测试函数并行执行
Error 失败并打印错误信息并继续执行，Fail+Log
Fatal 失败立即停止当前测试函数执行并打印错误信息，FailNow+Log

Linux系统编程——进程和线程

Posted on 2020-07-03 Edited on 2021-05-23 In Linux

进程

进程：程序在操作系统中执行时的一个实例，管理操作系统分派资源的基本单位。进程的两个基本要素是程序代码（要执行的指令）和相关数据集（进程局部变量，全局变量，申请的内存空间等），其中程序代码可以被多个进程共享（当进程派生出子进程的时候，该进程和该进程的子进程使用的是同一份程序代码）。进程有几个重要的特征，分别是：动态，并发，独立，异步

特征 说明
动态 程序存储在某种介质上的指令集合，是静态的。而进程则是程序在操作系统中执行的一个实例，它是创建而产生，调度而执行，撤销而消亡。一个进程从创建到消亡之后就不复存在了，所以，进程具有动态性
并发 在内存中通常存在多个进程，这些进程在一段时间内可以同时执行，这种现象叫并发
独立 进程之间是相互独立存在的，拥有操作系统分配给它们的资源，运行也是独立的
异步 各个进程之间没有特别控制的情况下按各自独立的，不可预知的速度执行。它们互不干扰，互不影响

进程控制块PCB
进程控制块PCB（process control block）指的包含一些和进程相关的数据结构，在进程切换状态的时候需要读写这个进程控制块。比如暂停一个进程的时候，为了防止数据丢失，需要将相关的数据写入进程控制块。当恢复一个暂停的进程继续执行的时候，需要读取这个进程控制块来将相关的信息恢复到暂停前的状态。

进程控制块的一些主要成员

名称 说明
进程ID 进程的唯一标识符，用于唯一标记一个进程，来区别于其它进程
状态 进程的状态，比如执行时为运行态，暂停时为阻塞态
程序计数器 包含下一个执行的指令地址
优先级 相对于其它进程的执行顺序，如果一个进程优先级高，那么这个进程有限执行
上下文 进程执行时处理器寄存器中的数据，比如栈指针，临时存放在寄存器等一些数据
I/O状态信息 包括I/O请求，分配给进程的I/O设备和被进程使用的文件列表等
记账信息 比如进程使用处理器时间总和，进程占用处理器时间片等信息
… …

那么进程暂停的时候进程控制块保存在哪里呢？在内存中还是虚拟内存中？运行态肯定是在内存中，那么阻塞态呢？

进程状态

从最直观的角度来看，一个进程在操作系统中要么处于运行状态，要么处于非运行状态。比如当一个进程被创建但是还没有运行就处于非运行状态，这个时候需要有一个存放非运行状态进程的容器，假设是一个队列。那么这个刚被创建的非运行状态进程就被放进这个队列里面。当正在运行的进程时间片到或者在等待一个事件发生，它就从运行态转变成非运行态，也被放进那个队列里面。这个时候处理器处于空闲状态，这就需要将某个处于非运行状态的进程调度到处理器执行。根据这个描述，就得到一个最简单的两状态模型。

树

Posted on 2020-07-03 Edited on 2021-05-23 In 数据结构

树不同于链表这样子的线性结构，只能一对一或者一对零。树是一种非线性结构，一个结点可以对应一个结点，也可以对应多个节点，甚至可以对应零个结点。在数据结构中，树的定义是：有n（n>=0）个节点的有限集合。当n等于0的时候称为空树，在一颗非空的树由如下特点：

  1. 有且仅有一个特定的，称为根的节点
  2. 当n>1的时候，其余节点可分为m（m>0）个互不相交的有限集合，每一个集合本身又是一棵树，称为子树

树的示意图

上图中，1是这棵树的根节点。根节点下有两个子节点，分别是2和3。没有子节点的节点称为树的叶子节点，在上图中，5，6，7，8，9这五个节点都没有子节点，所以是叶子节点。另外，根节点1的子节点2下面有很多个节点。它们共同的祖先都是2，如果以2为根节点，那么又是一棵树。称为根节点1的子树。当然，根节点1的另一个子节点3也可以作为子树的根节点，不过这棵树除了根节点3之外只有一个叶子节点。

从根节点到叶节点分成不同的层级，就像下图所示。树的层级从根节点开始，根节点所在的层级为第一层，它的直接子节点为第二层（如下图的2和3节点所在的层级），依次类推。所以，下图中的树一共有5层。对于一颗树的任意节点n，从根节点到n节点的唯一路径长称为n节点的深度。比如下图中的节点2的深度为1，10的深度为4。而根节点的深度为0。还是对于一棵树的任意节点n，以节点n为根节点的树，从n到属于这个树的叶子的最长路径长称为n节点的高度。比如节点2，它到叶子节点的最长路径长是3，也就是节点2到节点10的路径长。所以，所有叶子的高度都为0。因此，一棵树的高度等于它的最深叶子的深度。又是对于一个树的任意节点，它直接衍生出来的直接下一层级的节点称为该节点的子节点（除了叶子节点），比如节点4和5是节点2的子节点。而衍生出该节点的直接上一层级的节点称为该节点的父节点（根节点除外），比如节点1是节点2的父节点。同一层级具有共同父节点的节点称为兄弟节点，比如节点4和5有共同的父节点2，或者节点7，8，9有共同的父节点4，它们都是彼此兄弟节点。
