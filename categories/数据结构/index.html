树

树不同于链表这样子的线性结构，只能一对一或者一对零。树是一种非线性结构，一个结点可以对应一个结点，也可以对应多个节点，甚至可以对应零个结点。在数据结构中，树的定义是：有n（n>=0）个节点的有限集合。当n等于0的时候称为空树，在一颗非空的树由如下特点：

  1. 有且仅有一个特定的，称为根的节点
  2. 当n>1的时候，其余节点可分为m（m>0）个互不相交的有限集合，每一个集合本身又是一棵树，称为子树

树的示意图

上图中，1是这棵树的根节点。根节点下有两个子节点，分别是2和3。没有子节点的节点称为树的叶子节点，在上图中，5，6，7，8，9这五个节点都没有子节点，所以是叶子节点。另外，根节点1的子节点2下面有很多个节点。它们共同的祖先都是2，如果以2为根节点，那么又是一棵树。称为根节点1的子树。当然，根节点1的另一个子节点3也可以作为子树的根节点，不过这棵树除了根节点3之外只有一个叶子节点。

从根节点到叶节点分成不同的层级，就像下图所示。树的层级从根节点开始，根节点所在的层级为第一层，它的直接子节点为第二层（如下图的2和3节点所在的层级），依次类推。所以，下图中的树一共有5层。对于一颗树的任意节点n，从根节点到n节点的唯一路径长称为n节点的深度。比如下图中的节点2的深度为1，10的深度为4。而根节点的深度为0。还是对于一棵树的任意节点n，以节点n为根节点的树，从n到属于这个树的叶子的最长路径长称为n节点的高度。比如节点2，它到叶子节点的最长路径长是3，也就是节点2到节点10的路径长。所以，所有叶子的高度都为0。因此，一棵树的高度等于它的最深叶子的深度。又是对于一个树的任意节点，它直接衍生出来的直接下一层级的节点称为该节点的子节点（除了叶子节点），比如节点4和5是节点2的子节点。而衍生出该节点的直接上一层级的节点称为该节点的父节点（根节点除外），比如节点1是节点2的父节点。同一层级具有共同父节点的节点称为兄弟节点，比如节点4和5有共同的父节点2，或者节点7，8，9有共同的父节点4，它们都是彼此兄弟节点。

二叉树

所谓二叉树就是每个节点至多只能有两个子节点，也可以有一个子节点或者一个子节点都没有。根据树的定义，没有子节点的节点称为叶子节点。一棵二叉树如下图所示。其中，每个节点至多只有两个字节。比如，根节点下有两个子节点，分别是2和3。2被称为根节点的左儿子，3被称为根节点的右儿子。以2为根节点的子树称为根节点1的左子树。

散列表

基本概念

散列表

散列表也叫哈希表，是一种能够实现将多个键映射到多个值的数据结构，其添加，删除，查找等操作都是常数时间完成。其中，键和值的映射关系是通过散列函数来完成的。理想的散列表是一个固定大小的数组，不同的键经过散列函数的计算之后能够均匀分布到不同单元的散列表项当中，但这只是在理想的状态下。

散列函数

散列函数就是将任意长度的数据映射到固定长度数据的函数。散列表使用散列函数来计算键在散列表的索引，来找到键在散列表中对应的值。一个好的散列函数能够将不同的键均匀的映射到散列表的不同单元中，提高散列表利用率。所以，散列函数的好坏取决于其解决冲突的能力。但凡事都有利弊。一个好的散列函数通常会增加计算的开销，效率有所降低。所以，适当选择不是很完善的散列函数有助于提高效率。

取余法

最简单的散列函数是取余法，以键的长度对散列表的大小取余。对于键是字符串的形式，通常会先计算字符串的长度，然后根据字符串的长度对散列表的大小取余。一般来说，散列表的大小取素数会更好。取素数的散列计算方式：hash(key) = len(key) mod hashTableSize。如果散列表的长度不是素数，那么计算公式就是：hash(key) = (len(key) mod p) mod hashTableSize，p是一个大于hashTableSize的素数。

为什么取余法的散列表长度要取素数

简而言之就是为了减少冲突，使得键在散列之后能够均匀分布在散列表中

数组和链表

数组

数组是一种数据类型相同，长度固定，顺序存储的数据结构。也就说初始化的时候，会分配一块固定大小的内存，元素的存储时连续的。

优点

  1. 在下标范围内支持快速随机读取和更新数据，时间复杂度为O(1)

缺点

  1. 在插入和删除元素的时候需要移动元素位置。时间复杂度为O(n)
  2. 长度固定，想要存储更多元素的时候需要扩容

数组结构体声明
因为需要扩容，所以不直接使用数组，而是封装成一个结构体

栈和队列

栈

栈是一种先进后出的数据结构，入口和出口都在同一端，称为栈顶，所有操作都在栈顶进行。例如：进栈顺序是：1 2 3 4。出栈的顺序就是倒着过来：4 3 2 1。
栈模型

一个空栈模型
（这里的栈顶和栈底和操作系统里的栈顶和栈底不是一个意思，这里额栈顶是指入栈和出栈的一端，栈底是另一端）

用top指向最新的元素位置，因为这是一个空栈，所以top指向栈以外的地方

入栈（压栈）
