// 内存中并不是按照一个字节一个字节排列
// 类似于这样

         0   1   2   3   4   5   6   7
       +---+---+---+---+---+---+---+---+
data   |   |   |   |   |   |   |   |   |
       +---+---+---+---+---+---+---+---+

// 而是按照一个块一个块存取的
// 假设存取粒度是4，那么就像这样

         0   1   2   3       4   5   6   7
       +---+---+---+---+   +---+---+---+---+
data   |   |   |   |   |   |   |   |   |   |
       +---+---+---+---+   +---+---+---+---+

为什么要对齐

假设CPU存取粒度为4，要读取一个int32（4个字节）的变量，根据内存存取粒度为4的情况，每一个周期读取数据都要读取4个字节。假设现在没有内存对齐，并且这个int32类型的变量数据随意存放，假设它的存储起始地址是1，那么这个数据占据的内存地址就是1，2，3，4。现在要从0地址开始读取，一个周期读取4个字节，包括0，1，2，3。但是int32变量的数据还包括地址为4的数据，那么就需要进行迭第二个周期读取，读取包括4，5，6，7。然后将不需要的字节剔除，就是把0，5，6，7这个四个地址的字节剔除，最后剩下的1，2，3地址的数据块和地位是4的数据块组合得到int32变量的数据。

Golang之plan9汇编入门学习

简介

想研究研究golang的底层运行原理，涉及到plan9汇编。就想着怎么也得对plan9汇编了解了解，上网查了下plan9汇编的相关知识。然后…没错，老夫压根就看不懂！先记下来装个逼

plan9汇编的操作数方向，比如这个数据传送的方向是：第一个操作数的数据传送到第二个操作数

Plan9寄存器

通用寄存器

关键字 说明 备注
AX 累加寄存器 （感觉还可以存放数据）用于加、减、乘、输入输出等操作
BX 基址寄存器 可作为存储器指针使用
CX 计数寄存器 循环和字符串操作数，用于控制循环次数。输入输出操作时，用CL知名移位的位数
DX 数据寄存器 进行运算时，可作为操作数参与运算，也可以存放I/O端口地址
SI 源变址寄存器 存放源存储单元在段内的偏移量
DI 目的变址寄存器 存放目的存储单元在段内的偏移量
BP 基址指针寄存器 指向栈顶，低地址
SP 栈指针寄存器 指向栈底，高地址
PC 程序计数器 存放下一个指令的地址

一点补充

  1. 数据寄存器AX、BX、CX、DX用于保存操作数和运算结果。
  2. 变址寄存器SI、DI的作用是存放存储单元在段内的偏移量，可以根据：基地址加上偏移量来得到真正的物理地址
  3. 栈指针寄存器BP、SP主要用于存放栈内存储存单元的偏移量，也可以用于存储算术逻辑运算的操作数和结果

Golang并发调度

基于1.14版本

GMP模型

Go的调度器由三个部分组成，分别是G、M、P。G表示一个并发任务单元，M表示一个工作线程，P表示一个处理器。

runtime/proc.go

G

即goroutine，是一个并发任务单元。但是它并不是真正的执行体，它只保存任务的状态。真正的执行体是系统线程M。其实现是一个类似于PCB(进程控制块)的数据结构，其中包含为并发任务执行所需的栈空间，保存上下文的运行现场结构，goroutine的状态等等。在goroutine切换的时候，运行现场会保存当前goroutine的上下文，下次再恢复执行的时候就可以从中读取所需要的数据。一个goroutine创建之后，就会被放到P的本地队列或者全局队列当中，等待被调度执行。
